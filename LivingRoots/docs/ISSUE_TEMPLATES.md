# Issue Templates Documentation

## Overview
This document describes the issue templates created for the Living Roots project to ensure consistent and effective issue tracking.

## Available Templates

### 1. General Issue Template (`issue-template.md`)
A comprehensive template for creating all types of issues in the repository. Includes sections for:
- Type of issue (Bug Report, Feature Request, Refactoring Task, etc.)
- Description and context
- Expected vs. actual behavior
- Acceptance criteria
- Priority assessment
- Environment information (for bugs)
- Related issues

### 2. Bug Report Template (`bug-report-template.md`)
Specifically designed for reporting bugs in the Living Roots mod. Includes:
- Clear description of the bug
- Step-by-step reproduction instructions
- Expected vs. actual behavior
- Environment information (Stardew Valley version, SMAPI version, OS)
- Screenshots/logs section
- Additional context about impact on agroecological principles

### 3. Feature Request Template (`feature-request-template.md`)
Designed for suggesting new features, with consideration for the mod's agroecological philosophy. Includes:
- Problem description
- Proposed solution
- Alternatives considered
- Agroecological alignment section
- Acceptance criteria checklist
- Roadmap context

### 4. Pull Request Template (`pull-request-template.md`)
For submitting code changes, ensuring quality and consistency. Includes:
- Summary of changes
- Type of change classification
- Testing information
- Checklist for code quality, documentation, and testing
- Related issues linking
- Agroecological impact assessment

## Usage Guidelines

### For All Templates
- Be as specific as possible in descriptions
- Fill out all relevant sections
- Use the agroecological alignment sections when applicable
- Reference related issues when appropriate

### For Bug Reports
- Include exact steps to reproduce
- Specify the environment (game version, mod version, OS)
- Add screenshots when helpful

### For Feature Requests
- Explain how the feature aligns with agroecological principles
- Provide clear acceptance criteria
- Consider placement in the roadmap progression

### For Pull Requests
- Follow the project's architecture principles (SOLID, TDD, DDD)
- Include appropriate testing information
- Complete all checklist items before submission

## Benefits of Using Templates

1. **Consistency**: All issues follow similar structure for easier processing
2. **Completeness**: Templates help ensure all necessary information is provided
3. **Efficiency**: Reduces back-and-forth communication to clarify issues
4. **Alignment**: Keeps contributions aligned with the project's agroecological mission
5. **Quality**: Maintains high standards for code contributions through PR template

## Template Maintenance

These templates should be reviewed periodically and updated as the project evolves. Suggestions for template improvements can be made via issues using the general issue template.
